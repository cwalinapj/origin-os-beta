# Manifest Specs

This folder is for normative manifest specifications.

It should define required fields, canonical serialization rules, and compatibility/versioning behavior.

## Versioning notes

- `manifest-v1.md` describes the current stable `ManifestV1` contract.
- Manifest evolution and compatibility strategy are documented in:
  - `/docs/protocols/manifest-evolution.md`
  - `/docs/adr/0002-manifest-v1-scope-and-v2-evolution.md`
